# PBI-013 — Create PLS contract schema and persistence model

## Type
Task

## Parent

## Objective
Implement the PostgreSQL + Prisma persistence model for MVP PLS contracts using the restricted single-venture mudarabah baseline.

## In scope
- define persistence schema for `pls_contracts`
- define Prisma enums and model
- define PostgreSQL migration with constraints
- define required fields and nullability
- define persistence validation schema for required payload checks

## Out of scope
- REST controller
- service orchestration
- profit/loss computation
- settlement logic
- audit event generation
- Shariah workflow UI

## Acceptance criteria mapping

### AC1
Given ADR-2026-001 required fields, when the migration is applied, then the system can persist a PLS contract with:
- contract_id
- procurement_reference_id
- party roles
- capital_amount
- profit ratios
- restrictions
- review status
- timestamps

### AC2
Given an invalid contract payload, when required fields are missing, then persistence is rejected with validation errors.

## Design notes
- Use a single table for MVP.
- Model one `rabb_al_mal` and one `mudarib`.
- Model restrictions as JSON for flexibility.
- Store only `profit_sharing_ratio_rabb_al_mal` and `profit_sharing_ratio_mudarib`; do not store a redundant generic total ratio field.
- Enforce ratio-total rule in validation and database constraint.
